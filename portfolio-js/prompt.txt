\"I am using React, Framer Motion, and Tailwind CSS. Currently, my video modal is appearing at the very top of the screen and is cut off on mobile devices. I need to fix the modalVideo section to ensure the following:

Perfect Centering: The modal must be vertically and horizontally centered in the viewport on all screen sizes (fixed inset-0 with flex items-center justify-center).

Mobile Responsiveness: Ensure the video doesn't exceed the screen height or width. Use max-h-[90vh] and w-full to prevent the video from being cut off on small screens.

Stacking Context: Ensure the z-index is high enough to cover all other elements, including headers or sticky components.

Layout Fix: Please provide the updated JSX for the {modalVideo && (...)} block, ensuring the container uses h-screen or min-h-screen to properly calculate the center of the mobile display.

UX: Keep the 'Click outside to close' functionality and the 'X' close button."

The "Quick Fix" Code
If you want to apply the fix manually based on that logic, replace your current modalVideo block with this optimized version:

JavaScript
{modalVideo && (
  <motion.div 
    initial={{ opacity: 0 }}
    animate={{ opacity: 1 }}
    exit={{ opacity: 0 }}
    // Changed: Added h-screen and fixed positioning to ensure it fills the mobile viewport
    className="fixed inset-0 w-full h-screen bg-black/90 flex items-center justify-center z-[999] p-2 md:p-4" 
    onClick={closeModal}
  >
    <motion.div 
      initial={{ scale: 0.9, y: 20 }}
      animate={{ scale: 1, y: 0 }}
      className="relative w-full max-w-4xl max-h-[90vh] flex items-center justify-center" 
      onClick={e => e.stopPropagation()}
    >
      <button
        onClick={closeModal}
        className="absolute -top-12 right-0 md:top-4 md:right-4 text-white z-[1000] p-2 bg-white/10 hover:bg-white/20 rounded-full transition-colors"
      >
        <X size={24} />
      </button>
      
      <video
        src={modalVideo}
        controls
        autoPlay
        // Changed: Added max-h to prevent the video from taller than the phone screen
        className="w-full max-h-[80vh] md:max-h-[90vh] rounded-xl shadow-2xl border border-white/10"
      />
    </motion.div>
  </motion.div>
)}
Why this works:
z-[999]: Ensures it stays above your project cards.

h-screen: Specifically tells the browser to use the full mobile height for centering.

max-h-[80vh]: On mobile, videos are often aspect-ratio heavy; this prevents the video from pushing the modal boundaries off-screen.

-top-12 for Mobile X: Moves the close button above the video on small screens so it doesn't overlap the video content.
